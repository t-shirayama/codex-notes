# OpenAI Codexのベストプラクティス まとめ

[ベストプラクティス](https://developers.openai.com/codex/learn/best-practices)

基本的な考え方

Codexは「一回使い捨てのツール」ではなく、育てていくチームメンバーとして扱うのが効果的。

---

## 主なポイント

1. **プロンプトの書き方**
   タスクを依頼するときは「目標・背景・制約・完了条件」の4点を含めると精度が上がる。
2. **複雑なタスクは計画から**
   いきなりコードを書かせず、まず/planコマンドで計画を立てさせる。
3. **AGENTS.md を活用する**
   リポジトリのルール（ディレクトリ構成・テスト方法・コーディング規約など）をこのファイルに書いておくと、毎回説明しなくてよくなる。
4. **設定ファイルで一貫性を保つ**
   config.tomlに個人・プロジェクトごとのデフォルト設定を書いておく。
5. **テストとレビューも任せる**
   コードを書かせるだけでなく、テスト・Lint・差分レビューまでセットで依頼する。
6. **MCP で外部ツールと連携**
   GitHubやJiraなど外部システムのデータが必要なときはMCPで接続する（コピペ不要になる）。
7. **繰り返し作業はスキル化**
   同じプロンプトを何度も使うならSKILL.mdにまとめて再利用できるようにする。
8. **安定したワークフローは自動化**
   定期的なタスク（リリースノート作成・CI確認など）はAutomation機能でスケジュール実行できる。

## よくある失敗

- 毎回プロンプトに同じルールを書く（→ AGENTS.mdに移すべき）
- 複雑なタスクをいきなり実行させる（→ 計画フェーズを挟む）
- 不安定なうちに自動化する（→ まず手動で安定させてから）
- 1プロジェクト = 1スレッドにする（→ 1タスク = 1スレッドが正解）
