# 最初の1セッションの進め方

## 概要

Codexを最初から上手く使うコツは、いきなり大きい依頼を投げずに、1セッション1目的で進めることです。

## 基本の流れ

### 1. 目的を1つに絞る

- 良い例: `ログイン画面の文言だけ直したい`
- 悪い例: `認証、UI、CI、READMEを全部整理したい`

### 2. 対象を明示する

- 関連ファイルを開く
- `@ファイル名` で参照を追加する
- 期待する完了条件を書く

### 3. 複雑なら `/plan` を使う

要件が曖昧なときは、いきなり実装させずに計画を先に作ると失敗が減ります。

## 依頼テンプレート

```txt
目的:
トップページの見出しだけ修正したい

対象:
@src/app/page.tsx

制約:
既存のデザインは変えない

完了条件:
見出し文言を変更し、差分を説明する
```

## よくある失敗

- 背景説明だけ長くて、変更対象が書かれていない
- 1スレッドで別問題を混ぜる
- 完了条件がなく、どこで終わるか曖昧

## 具体例

「このエラーを直して」だけではなく、「`@api.ts` の 401 エラー処理を修正し、再発防止のテスト案も出して」と書くと、Codexが動きやすくなります。
