# アーキテクチャ概要

## 概要

Webアプリ全体の画面、API、DB、外部サービスの関係を書く。

## 主要領域

- UI: ログイン状態、主要画面、エラー表示
- API: 認証状態取得、ログイン開始、ログアウト
- DB: ユーザー、セッション、監査ログ
- 外部サービス: 認証プロバイダー、メール送信、分析

## データの流れ

1. 利用者が画面を開く
2. UIが認証状態を確認する
3. APIがセッションまたは認証プロバイダーを確認する
4. 結果に応じてログイン表示またはユーザー情報を返す

## 変更時の注意

- APIを増やしたら `api.md` を更新する
- テーブルや保存項目を増やしたら `data-model.md` を更新する
- 外部サービスを増やしたら `environment.md` と `.env.example` を更新する
