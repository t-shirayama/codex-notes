# story-001-auth 要件

## 概要

利用者がログインし、ログイン状態に応じて画面表示が変わる導線を作る。

このstoryは認証プロバイダーの選定や本番用セキュリティ設計を決めるものではなく、Webアプリ開発で影響範囲が広い認証導線を例として扱います。

## ユーザーストーリー

利用者として、自分がログインしているかを画面上で確認したい。未ログインの場合はログインへ進み、ログイン済みの場合は自分の表示名を確認したい。

## 受け入れ条件

- 未ログイン時はログインボタンが表示される
- ログイン済み時は表示名またはメールアドレスが表示される
- ログイン失敗時は再試行できるエラー表示が出る
- ログアウト後は未ログイン表示に戻る
- 認証状態の取得に失敗した場合の扱いがplanに書かれている

## 対象外

- 本番用の認証プロバイダー選定
- 多要素認証
- 管理者ロールの詳細設計
