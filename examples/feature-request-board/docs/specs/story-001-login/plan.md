# story-001-login 計画

## Goal

ログイン状態の表示導線を追加するために、要件、画面変更、レビュー観点を整理する。

## Steps

1. 既存のヘッダー構成を確認する
2. 未ログイン時とログイン済み時の表示を決める
3. エラー時の表示方針を決める
4. 受け入れ条件とレビュー観点を照合する

## Acceptance Criteria

- `requirement.md` の受け入れ条件がすべてplanに反映されている
- 画面変更の範囲がヘッダー周辺に限定されている
- レビュー時に確認すべき観点が `docs/guidelines/review.md` から辿れる

このファイルは、完了後に削除または `docs/archive/plans/` へ移すルート `PLAN.md` ではなく、story単位の作業計画例です。
