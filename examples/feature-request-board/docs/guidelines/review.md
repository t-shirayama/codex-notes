# レビュー観点

## 目的

Feature Request Board の例で、Codexにレビューを依頼するときの観点を揃える。

## チェックリスト

- 要件とplanの受け入れ条件が矛盾していないか
- 未ログイン時とログイン済み時の表示が両方考慮されているか
- 対象外の実装に踏み込んでいないか
- `context.json` の references が最新か

## レビュー依頼例

```txt
/review
examples/feature-request-board の story-001-login をレビューして。
要件、plan、review.md の整合性を優先して見て。
```
