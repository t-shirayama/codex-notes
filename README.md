# codex-notes

Codexを日常運用で迷わず使うための、個人用の最小ハンドブックです。
網羅的なナレッジベースではなく、繰り返し使う判断基準だけを残します。

## 読む順番

1. [Codex運用ハンドブック](./docs/handbook.md) で、依頼、計画、確認、再利用の基本方針を読む
2. [Codex仕様リファレンス](./docs/reference.md) で、VS Code / CLI / AGENTS.md / MCP / skills / hooks の確認先を見る

## 残すもの

- `docs/handbook.md`: 自分のCodex運用ルール
- `docs/reference.md`: 公式情報への短い参照メモ
- `AGENTS.md`: このリポジトリを編集するときの最小ルール

## 運用方針

- 重複記事を増やさず、重要な判断基準は `docs/handbook.md` に統合する
- 仕様寄りの情報は `docs/reference.md` に寄せる
- 参考元は各記事の末尾にMarkdownリンクで残す
- 共有skill、hookスクリプト、実設定ファイルは同梱しない
