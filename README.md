# codex-memo

VSCodeでCodex（OpenAIのコーディングエージェント）を初めて使う人向けのメモ。

---

## セットアップ

### 1. 拡張機能のインストール

VSCode の拡張機能マーケットプレイスで `Codex` を検索し、OpenAI が公式公開している **Codex – OpenAI's coding agent** をインストールする。

### 2. サインイン

インストール後、右サイドバーに Codex パネルが表示される。**Sign in with ChatGPT** ボタンからサインイン。利用には ChatGPT Plus 以上のプランが必要。

> Windowsの場合はWSL2環境での利用を推奨。ネイティブでも動くが、Linux系コマンドとの相性が良い。

---

## AGENTS.md の設定

AGENTS.md はCodexへの永続的な指示書。タスク開始前に毎回読み込まれるため、プロジェクトの規約やコーディングスタイルをここに書いておくと一貫した動作になる。

### 読み込まれる場所（優先度順）

| 場所                           | 用途                               |
| ------------------------------ | ---------------------------------- |
| `~/.codex/AGENTS.md`           | 全プロジェクト共通のグローバル設定 |
| リポジトリルートの `AGENTS.md` | プロジェクト全体の規約             |
| サブディレクトリの `AGENTS.md` | 特定機能・チーム向けの上書き設定   |

複数ファイルが存在する場合は上から順にマージされて読み込まれる。

### 書き方の例

```markdown
# AGENTS.md

## 言語設定

- 返答はすべて日本語で行うこと

## コーディング規約

- Python のパッケージ管理は pip ではなく uv を使うこと
- 変更後は必ず `pytest` を実行してテストが通ることを確認すること
- 新しい本番依存を追加する前は確認を求めること

## コミット規約

- コミットメッセージは Conventional Commits に従うこと（例: `feat:`, `fix:`, `docs:`）
```

## 基本的な使い方

### モードの選択

Codexには3つのモードがあり、タスクに応じて切り替える。

| モード                  | できること                                               | 向いている場面                   |
| ----------------------- | -------------------------------------------------------- | -------------------------------- |
| **Chat**                | 会話・相談のみ。ファイル操作なし                         | 設計の相談、コードレビューの依頼 |
| **Agent**               | ファイルの読み書き・コマンド実行が可能。実行前に確認あり | 通常の開発作業（推奨）           |
| **Agent (Full Access)** | 確認なしで自律的に実行                                   | 使い捨てのブランチや実験的な作業 |

> 普段の開発は **Agent** モードで運用し、重要なリポジトリで **Agent (Full Access)** は使わないようにするのが無難。

### Reasoning Effort の調整

**Low / Medium / High** の3段階で切り替えられる。

- **Low**: 速く・シンプルなタスク向け
- **Medium**: デフォルト。ほとんどのタスクで十分
- **High**: 複雑な問題や大規模なリファクタリング向け（処理に時間がかかる）

### プロンプトのコツ

- 対象ファイルを開いておくか `@ファイル名` で明示するとより精度が上がる
- タスクはなるべく小さく・具体的に伝える
- 「〇〇を修正して、テストも追加して」のように一度に伝えてよい

## よく使う操作

### コンテキストの指定

| 操作                           | 方法                                                 |
| ------------------------------ | ---------------------------------------------------- |
| 特定のファイルを参照させる     | チャット入力欄で `@ファイル名` と入力                |
| 選択範囲を対象にする           | コード選択 → Codexパネルに質問                       |
| 開いているファイルを参照させる | ファイルを開いたままタスクを依頼（自動で参照される） |

### Cloud モード（長時間タスクの委譲）

処理に時間がかかる作業はクラウドに委譲できる。

1. セッションタイプのドロップダウンで **Cloud** を選択
2. タスクを入力して実行
3. 進捗はIDEから確認でき、完了後にdiffをローカルに適用できる

### キーボードショートカット

ショートカットは設定から変更・確認できる。

VSCode の設定アイコン → **Keyboard Shortcuts** → `Codex` で検索。

よく使うアクション（デフォルト設定）：

| アクション                   | ショートカット（Mac） |
| ---------------------------- | --------------------- |
| Codexチャットを開く/閉じる   | 設定から割り当て可能  |
| 選択範囲をコンテキストに追加 | 設定から割り当て可能  |
