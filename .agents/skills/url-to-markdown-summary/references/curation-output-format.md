# Curation Output Format

Qiita、Zenn、ブログ記事を再編集するときは、記事単位の感想ではなく、テーマ単位のMarkdownに再構成する。

```md
# タイトル

> 種別: curation
> 参考元: [記事名](URL)
> 最終ファクトチェック: YYYY-MM-DD
> 確認メモ: 主張と要点を確認

## 概要

対象記事群から何を学べるかを短く書く。

## 主な知見

- 知見1
- 知見2
- 知見3

## 実務で参考になる点

### 1. すぐ使える工夫

再利用しやすいポイントを書く。

### 2. 前提が分かれる点

人や環境によって変わる点を書く。

### 3. 公式情報との関係

公式と一致する点、補足として有効な点を書く。

## 注意点

- 一般化しすぎない
- 前提条件が強い場合は明記する

## 参考元

このページは以下の参考元に基づいて整理しています。未確認の推測は含めていません。

- URL 1
- URL 2
```

## Notes

- 記事ごとの要約を並べない
- 主張より再利用できる知見を優先する
- 公式一次情報があるなら、必要に応じて参照を補う
