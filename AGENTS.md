# AGENTS.md

## Purpose

このリポジトリは、自分がCodexを使うときに迷わないための最小ハンドブックです。
AIは、網羅性よりも「少ない文書で再利用しやすいこと」を優先してください。

## Editing Rules

- 重要な運用判断は `docs/handbook.md` に統合する
- 公式仕様や確認先は `docs/reference.md` に寄せる
- 同じ内容の新規記事を作らない
- 旧記事を分割して増やさず、重複は削除または統合する
- 目次や主要導線を変えたら `README.md` も更新する

## Source Priority

1. 公式ドキュメント
2. 実測や一次情報
3. Qiita / Zenn / ブログ記事を再編集した知見

推測で断定しないでください。外部情報を使う場合は、参考元をMarkdownリンクで本文末尾に残します。

## Repository Policy

- 共有skillやhookスクリプトは同梱しない
- `.codex` や開発環境の実設定は同梱しない
- 参考元管理は記事台帳ではなく、各統合記事の末尾に集約する
- テンプレートファイルは同梱しない

## Verification

- 削除済み構成へのリンクが残っていないか `rg` で確認する
- `find docs -type f | sort` で、意図したファイルだけが残っているか確認する
- 変更後は `git status --short` で削除と更新の範囲を確認する
